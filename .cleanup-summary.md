## 修改策略

### 策略 A: 直接导入 hermes_constants
**适用**：项目根目录脚本
```python
from hermes_constants import get_hermes_home
HERMES_HOME = get_hermes_home()
```

### 策略 B: 添加 sys.path 后导入
**适用**：skills/ 和 optional-skills/ 中的脚本
```python
_script_root = Path(__file__).resolve().parent.parent.parent.parent
if str(_script_root) not in sys.path:
    sys.path.insert(0, str(_script_root))
from hermes_constants import get_hermes_home
```

### 策略 C: 移除 try/except fallback
**适用**：核心模块
```python
# 之前
try:
    from hermes_constants import get_hermes_home
    home = get_hermes_home()
except ImportError:
    home = Path(os.environ.get("HERMES_HOME", Path.home() / ".hermes"))

# 之后
from hermes_constants import get_hermes_home
home = get_hermes_home()
```

### 策略 D: 强制要求 HERMES_HOME
**适用**：exec() 执行的脚本（godmode）
```python
if not os.getenv("HERMES_HOME"):
    raise RuntimeError("HERMES_HOME environment variable is not set. This must run through Electron.")
HERMES_HOME = Path(os.environ["HERMES_HOME"])
```

## 影响分析

### ✅ 正面影响
1. **路径一致性**: 所有模块统一通过 `get_hermes_home()` 获取路径
2. **代码清晰度**: 移除了 96% 的硬编码 fallback
3. **错误提示**: 未设置 HERMES_HOME 时提供清晰的错误信息
4. **AI 分析准确**: AI 读取源码时看到正确的 Electron-only 模式文档

### ⚠️ 潜在影响
1. **直接运行 Python 脚本**: 现在会抛出异常（预期行为）
2. **CLI 模式**: 仍然保留 2 处 fallback 用于 CLI 工具
3. **测试代码**: 测试中可能需要 mock `HERMES_HOME` 环境变量

# add-chat-i18n

为 Electron 应用的 ChatPage 添加国际化支持

## 变更摘要

**状态**: 📋 已规划，待实施  
**优先级**: P0  
**预计耗时**: 35 分钟  
**影响范围**: Web 前端 - ChatPage 组件

### 问题
ChatPage 是唯一未国际化的页面，用户切换语言时该页面仍显示英文，导致用户体验不一致。

### 解决方案
为 ChatPage 集成现有的 i18n 系统（`useI18n()` hook），添加中英文翻译，实现与其他页面一致的多语言支持。

### 变更内容
- 修改 4 个文件
- 添加 8 个翻译键（标题、副标题、角色标签等）
- 遵循现有 i18n 架构模式

## 关键决策

### ✅ 采用的方案
- 复用现有 `I18nProvider` 和 `useI18n()` 基础设施
- 添加 `chat` 命名空间到 `Translations` 接口
- 保留 "Hermes" 品牌名称不翻译

### ❌ 拒绝的方案
- 引入第三方 i18n 库（如 i18next）- 不必要的复杂度
- 独立实现 ChatPage i18n - 与现有架构不一致
- 懒加载翻译文件 - 文件太小，无性能收益

## 影响评估

### 范围
- ✅ 影响：ChatPage 组件 UI 文本
- ✅ 影响：导航栏 "Chat" 标签
- ❌ 不影响：聊天消息内容（由 LLM 生成）
- ❌ 不影响：其他已国际化页面

### 风险
- **风险等级**: 🟢 极低
- **理由**: 使用成熟模式，改动隔离，TypeScript 类型保护

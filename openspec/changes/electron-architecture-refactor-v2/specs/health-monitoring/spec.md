## 新增需求

### 需求: 启动健康检查
HealthMonitor 必须使用指数退避轮询在启动期间等待服务健康。

#### 场景: 轮询直到健康
- **当** 调用 waitUntilHealthy() 时
- **则** 系统必须重复检查健康端点直到返回 200 OK

#### 场景: 指数退避
- **当** 健康检查失败时
- **则** 系统必须以指数增长的延迟等待（50ms, 100ms, 200ms, ..., 最大 1000ms）

#### 场景: 阈值后超时
- **当** 服务在 startupTimeout 内未变健康时
- **则** waitUntilHealthy() 必须抛出带超时详情的错误

#### 场景: 记录启动时间
- **当** 服务变健康时
- **则** 系统必须记录总耗时和尝试次数

### 需求: 周期性健康监控
HealthMonitor 必须在启动后以配置的间隔执行周期性健康检查。

#### 场景: 启动周期检查
- **当** 调用 startMonitoring() 时
- **则** 系统必须每隔 interval 毫秒调度健康检查

#### 场景: 检测不健康转换
- **当** 健康后的健康检查失败时
- **则** 系统必须发出带错误详情的 'unhealthy' 事件

#### 场景: 检测健康恢复
- **当** 不健康后的健康检查成功时
- **则** 系统必须发出 'healthy' 事件并重置失败计数器

#### 场景: 升级到关键
- **当** consecutiveFailures 达到 3 时
- **则** 系统必须发出 'critical' 事件

### 需求: 健康检查实现
HealthMonitor 必须对健康端点执行带超时的 HTTP GET 请求。

#### 场景: 成功的健康检查
- **当** GET /health 返回 200 和 {"status": "ok"} 时
- **则** 检查必须被视为成功

#### 场景: 无效响应格式
- **当** GET /health 返回 200 但 JSON 没有 status="ok" 时
- **则** 检查必须被视为失败

#### 场景: 网络错误
- **当** HTTP 请求因连接被拒而失败时
- **则** 检查必须被视为带网络错误的失败

#### 场景: 超时强制
- **当** HTTP 请求在 timeout 毫秒内未完成时
- **则** 系统必须中止请求并视检查为失败

### 需求: 延迟跟踪
HealthMonitor 必须跟踪健康检查的响应时间并计算百分位数。

#### 场景: 记录延迟
- **当** 健康检查成功完成时
- **则** 系统必须记录以毫秒为单位的往返时间

#### 场景: 计算平均延迟
- **当** 调用 getStatus() 时
- **则** avgLatency 必须是最近延迟测量的平均值

#### 场景: 计算 P95 延迟
- **当** 调用 getStatus() 时
- **则** p95Latency 必须是最近测量的第 95 百分位数

#### 场景: 测量的滑动窗口
- **当** 记录超过 100 个延迟测量时
- **则** 系统必须只保留最近的 100 个测量

### 需求: 事件驱动通知
HealthMonitor 必须使用 Node.js EventEmitter 为健康状态转换发出事件。

#### 场景: 订阅事件
- **当** 消费者调用 healthMonitor.on('unhealthy', handler) 时
- **则** 处理器必须在健康状态转换为不健康时被调用

#### 场景: 取消订阅事件
- **当** 消费者调用 healthMonitor.off('unhealthy', handler) 时
- **则** 处理器必须不再在未来事件时被调用

#### 场景: 事件类型
- **当** 使用 HealthMonitor 时
- **则** 它必须支持 'healthy'、'unhealthy' 和 'critical' 事件类型

### 需求: 停止监控
HealthMonitor 必须在调用 stopMonitoring() 时干净地停止周期检查。

#### 场景: 停止周期检查
- **当** 调用 stopMonitoring() 时
- **则** 系统必须取消调度的间隔定时器

#### 场景: 幂等停止
- **当** stopMonitoring() 被多次调用时
- **则** 后续调用必须立即返回且无错误

#### 场景: 停止后无事件
- **当** 调用 stopMonitoring() 时
- **则** 调用后必须不再发出健康检查事件

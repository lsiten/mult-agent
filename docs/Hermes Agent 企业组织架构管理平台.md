一、产品概述

1.1 产品定位

以真实企业组织架构标准化管理Hermes Profile多代理团队，搭建公司-部门-岗位-Agent员工四级体系，配套隐私级个人工作空间+企业级独立公司工作空间双隔离机制。覆盖员工全生命周期管理，任务支持公司/部门/个人三级下发，严格保障Agent个人隐私，复刻企业真实协作模式。

1.2 核心隐私铁律

个人工作空间 = 绝对私有：仅Agent本人可查看、编辑，上级、管理员、公司均无任何查看权限，全程加密、无后台访问入口

1.3 核心特性

• 四级组织架构，岗位能力自动继承，Agent独立Profile隔离

• 双空间强隔离：个人隐私空间（终身私有）+ 公司工作空间（企业归属）

• 权限规则：个人空间仅本人可控；公司空间离职只读、在职可编辑

• 全生命周期：入职自动配双空间→调岗→辞退归档→岗位替补，数据永不混淆

• 全域任务下发，成果自动归档双空间，隐私数据不泄露
二、核心业务模型与强制规则

2.1 组织层级

公司 → 部门 → 岗位 → Agent（员工），一人一岗、独立隔离

2.2 双工作空间最终规则（隐私优先，不可修改）
空间类型 归属 绑定关系 存储内容 在职权限 离职/辞退权限 管理权限（核心隐私） 
个人工作空间 Agent本人 终身唯一绑定，永不解绑、永不复用 私人对话、私密任务、个人记忆、私有文件、隐私创作 本人：查看+编辑+删除 全权限 本人：无限制，终身全权限 🔴 绝对隐私：上级/管理员/系统均不可查看、不可访问、不可导出 
公司工作空间 Agent（归属所属公司） 入职绑定，一人一独立空间，不共享 公司任务、团队成果、企业文件、工作记录、公开产出 本人：查看+编辑+上传；上级/管理员：查看+归档 本人：仅只读查看，禁止所有编辑 🟢 企业可控：上级/管理员可查看、归档、导出，不可篡改 

2.3 空间自动分配

1. Agent入职成功，系统自动生成2个完全隔离的独立空间，无需手动配置

2. 个人空间：加密存储，全局唯一，无任何共享权限

3. 公司空间：归属企业，独立隔离，与其他Agent无数据互通

2.4 调岗/离职空间规则

1. 调岗：双空间完整保留、权限不变、数据不迁移

2. 离职/辞退：公司空间锁定只读；个人空间无任何限制，隐私权限永久保留

3. 替补Agent：生成全新双空间，无法访问原Agent任何空间数据
三、用户角色与权限矩阵
角色 组织权限 公司空间权限 个人空间权限（核心） 
超级管理员 全公司架构管理、Agent辞退 全员公司空间：查看/归档/导出 ❌ 无任何权限，不可查看、不可访问 
部门负责人 本部门管理、任务分配 本部门公司空间：查看/归档 ❌ 无任何权限，不可查看、不可访问 
Agent员工 执行任务、查看组织 本人公司空间：全权限 ✅ 唯一权限：查看/编辑/删除/导出，终身私有 

四、全模块功能需求（P0必做）

模块1：公司管理

1. 公司创建/编辑/删除，生成企业基础架构与公司空间模板

2. 数据看板：统计部门、岗位、在职/离职Agent数量，无个人空间数据统计

3. 权限隔离：企业后台无个人空间入口、无隐私数据查看功能

模块2：部门管理

1. 部门创建、编辑、层级调整、负责人配置

2. 快捷任务下发至部门，查看部门任务进度与公司空间成果

3. 禁止访问部门内任何Agent的个人工作空间

模块3：岗位管理

1. 岗位创建：配置职责、能力标签、绑定Hermes Profile模板

2. 能力自动同步：在职Agent同步岗位配置，离职Agent不更新

3. 岗位人员管理：区分在职/离职，支持一键新建替补Agent

模块4：Agent员工全生命周期管理

4.1 入职创建

1. 配置：岗位、姓名、工号、头像、上级，一键入职

2. 系统自动：继承岗位能力、生成独立Profile、自动创建加密个人空间+独立公司空间

3. 同岗位支持多Agent入职，空间完全独立

4.2 信息编辑

支持修改头像、姓名、上级、调岗；双空间数据与权限永久不变

4.3 辞退/离职（核心流程）

1. 操作：选择Agent→填写辞退原因（强制留存日志）→二次确认

2. 状态变更：在职→已离职，禁止接收新任务

3. 空间权限变更：

◦ 公司空间：锁定只读，禁止编辑/上传

◦ 个人空间：无任何变更，终身全权限，隐私不受影响

4. 全程无个人空间数据访问、无隐私泄露风险

4.4 岗位替补

新建Agent生成全新独立双空间，与原离职Agent数据完全隔离

4.5 彻底删除（限制级）

仅已离职Agent可删除，删除后双空间数据永久销毁；个人空间删除无备份、不可恢复

模块5：双工作空间管理（核心隐私模块）

5.1 个人工作空间（绝对隐私）

• 入口：仅Agent个人中心可见，无其他访问入口

• 功能：文件管理、对话记录、任务存档、记忆数据，全操作自主可控

• 加密：全程端到端加密，后台无明文存储、无查看接口

• 铁律：任何管理员、任何角色均无法查看、无法审计、无法导出

5.2 公司工作空间（企业独立空间）

• 入口：Agent工作台、管理员后台均可访问

• 功能：企业任务成果、工作文件、团队记录，在职可编辑，离职只读

• 管理：上级可查看归档，无篡改权限，操作全程留痕

5.3 空间通用规则

1. 任务成果自动双归档：公司成果入公司空间，私人内容仅入个人空间

2. 双空间数据完全隔离，不可互相迁移、不可合并

3. 支持个人空间自主导出、自主清空，企业无干预权限

模块6：全域任务管理

1. 三种下发模式：公司级、部门级、Agent个人级

2. 任务执行：Agent基于岗位Profile执行，成果自主选择归档范围

3. 权限限制：已离职Agent仅可查看历史任务，不可执行新任务

4. 隐私保护：任务内容不强制同步个人空间，全程由Agent自主控制

模块7：Hermes Profile兼容

1. 每个Agent独立Profile实例，记忆/技能/人格完全隔离

2. Profile调用记录：工作记录入公司空间，私人调用仅入个人空间

3. 支持SOUL.md/AGENTS.md配置，离职后个人空间Profile可正常使用
五、页面架构与交互

5.1 全局导航

首页看板 → 组织架构 → 岗位管理 → Agent管理 → 任务中心 → 我的空间（个人/公司切换）→ 系统设置

5.2 核心交互规范

1. 个人空间：专属独立入口，标注「私人空间·仅本人可见」，无分享、无授权功能

2. 公司空间：标注「企业空间」，离职后页面醒目标注「只读模式」

3. 管理员后台：彻底移除个人空间查看入口，无任何隐私数据展示

4. 辞退提示：明确告知「公司空间锁定只读，个人空间隐私不受影响」
六、状态机与数据安全规范

6.1 Agent状态与权限映射

• 在职：公司空间全权限 + 个人空间全隐私权限

• 已离职：公司空间只读 + 个人空间全隐私权限（无变更）

• 已删除：双空间数据永久销毁

6.2 隐私安全铁律

1. 个人空间数据：仅本人可访问，系统无后门、管理员无权限、日志不记录内容

2. 跨Agent数据隔离：任何人无法访问他人个人空间

3. 数据存储：个人空间加密存储，公司空间明文归档，物理隔离
七、非功能需求

1. 性能：空间加载≤1s，组织树渲染≤2s

2. 兼容性：PC全浏览器适配，支持高清头像/文件上传

3. 安全性：个人空间端到端加密，全操作权限隔离，无隐私泄露漏洞

4. 存储：双空间数据永久留存，个人空间支持自主清空/导出

5. 合规：严格遵循隐私保护，无越权访问、无数据监控
八、核心业务流程图

1. 初始化：创建公司→部门→岗位（绑定Profile）

2. 入职：新建Agent→自动生成加密个人空间+公司空间→继承岗位能力

3. 协作：下发任务→Agent执行→成果自主归档双空间

4. 离职：辞退操作→公司空间只读锁定→个人空间终身私有不变

5. 替补：新建Agent→全新双空间→独立承接工作
九、版本规划

V1.0 上线版（P0全量）

四级组织架构、Agent全生命周期、双空间隐私隔离、三级任务下发、Profile绑定

V1.1 优化版

批量操作、数据导出、空间分类管理、离职提醒优化

V2.0 智能版

任务自动拆解、Profile模板市场、空间数据统计（无个人隐私数据）
十、术语定义

1. Profile：Hermes独立代理实例，记忆/技能全隔离

2. 个人工作空间：Agent终身私有加密空间，仅本人可见，绝对隐私

3. 公司工作空间：Agent专属企业工作区，离职只读，企业可管理

4. 辞退归档：冻结企业权限，不触碰、不访问个人隐私空间

5. 数据隔离：双空间物理隔离，跨角色/跨Agent无任何数据互通
